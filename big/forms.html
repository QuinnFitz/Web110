Forms


Forms are incredibly important for any website. They are one of the primary ways for a user to send data back to the server, allowing a dialogue to open up with potential clients, fans, etc.

As far as constructing forms, there are a lot of rules on how to, and how not to construct them. For instance, it’s considered taboo to nest a form inside another form. This can make the form behave in strange and unpredictable ways. However, you can use the fieldset element to put a group of widgets inside of a form. The label element is often used to label said widgets in a form.

To structure a form, it is common to wrap a widget and its label in a list element. Either a <ul> or <ol> list. This helps to structure groups of checkboxes and radio buttons. In addition to a <fieldset> element, headings and section elements also help to organize complicated forms.

It’s a common practice to style your HTML forms with CSS, so that they look more professional. When doing this, however, you have to keep a few things in mind. For instance, some HTML elements are more compatible with CSS than others. The above mentioned elements all work great with CSS(with the exception of radio buttons). As do elements like <label> and <output>, and many others. Some elements that may give you some trouble are-
<input type=”color”>
<input type=”range”>
<select>, <option>, <optgroup>, <datalist>
<progress>, <meter>
Data related controls like <input type=”datetime-local”>
Styling any of these elements using CSS would be quite difficult. Something that’s very convenient to style with CSS are fonts. ALthough, some browsers are compatible with more fonts than others. Overall, forms are a huge part of any website, allowing a dialogue to open with you and your client. And while they have a lot of structuring rules, and can be challenging to style, they are well worth the effort.
